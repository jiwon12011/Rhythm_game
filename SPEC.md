# 마요나카☆아쿠아비트 (MIDNIGHT AQUA BEAT) — 구현 스펙 v1.0

> **이 문서의 목적**: 이 문서 하나만 읽고 게임 전체를 구현할 수 있도록 작성된 마스터 스펙이다.
> 컨셉/세계관 프레젠테이션 버전은 `gdd.html` 참고. 두 문서가 충돌하면 **이 문서(SPEC.md)가 우선**한다.
> 구현자(사람 또는 AI)는 이 문서에 명시되지 않은 사항을 임의로 크게 바꾸지 말고, §14의 "재량 허용 범위"를 따를 것.

## 0. 한 줄 요약

**가로 16:9 웹 리듬게임 + 캐릭터 수집.** 폐수족관 라이브하우스를 배경으로, 수집한 밴드 멤버의 "컨셉포토"가 플레이 배경이 되고, 콤보가 쌓일수록 배경이 밝아진다. 판정·연출·수집의 모든 수치가 이 문서에 정의되어 있다.

## 1. 기술 스택 & 프로젝트 구조

### 1.1 스택 (고정)

- **순수 HTML + CSS + Vanilla JS (ES2020+)**. 빌드 도구, 프레임워크, 외부 라이브러리 **금지**.
- 게임플레이 렌더링: **Canvas 2D** (단일 캔버스). UI 화면(타이틀·준비·결과 등)은 DOM.
- 사운드: **Web Audio API** (§7). 프로토타입 곡은 신스 합성, 파일 로딩 없음.
- 저장: `localStorage` (§10).
- 로컬 파일(`file://`)로 열어도 동작해야 한다 → `fetch()`로 로컬 JSON을 읽지 말 것. 모든 데이터는 JS 파일로 포함.

### 1.2 파일 구조 (고정)

```
oshi-beat/
├── index.html            # 진입점. DOM 화면 골격 + 스크립트 로드
├── css/style.css         # 전체 스타일
├── js/
│   ├── data/
│   │   ├── characters.js # 캐릭터 12명 데이터 (§9 스키마, §5 내용)
│   │   └── songs.js      # 곡 + 채보 데이터 (§8 스키마)
│   ├── save.js           # localStorage 래퍼 (§10)
│   ├── audio.js          # 신스 엔진 + 스케줄러 (§7)
│   ├── game.js           # 리듬 코어: 노트, 판정, 점수, 캔버스 렌더 (§6)
│   ├── screens.js        # 화면 전환 + 각 화면 로직 (§4)
│   └── main.js           # 초기화, 리사이즈, 전역 상태
└── assets/
    ├── photos/           # 컨셉포토: {charId}_{n|r|e}.webp  (16:9)
    ├── chars/            # {charId}_full.png(전신), {charId}_face_{joy|fire|comfort}.png
    ├── covers/           # {songId}.webp (1:1 앨범아트)
    └── logos/            # {bandId}.png
```

### 1.3 에셋 부재 시 폴백 (중요)

이미지 에셋은 **나중에 사용자가 직접 생성해 넣는다.** 따라서 모든 이미지는 로드 실패 시 자동 폴백되어야 한다:

- 컨셉포토 없음 → 해당 캐릭터의 `color`를 사용한 그라데이션 배경 + 중앙에 캐릭터 이름 워터마크 텍스트(투명도 15%).
- 전신/표정 없음 → 캐릭터 `color` 원형 배지 + 이름 첫 글자.
- 앨범아트 없음 → 밴드 컬러 사각형 + ♪ 심볼.
- 폴백은 "깨진 이미지"처럼 보이면 안 되고, 의도된 디자인처럼 보여야 한다.

### 2.2 스타일 규칙

- 도트 그리드 배경 패턴 (`radial-gradient` 반복, 26px 간격)을 UI 화면 바탕에 사용.
- **금지**: 레인보우/무지개 그라데이션, 이모지를 장식 아이콘으로 남용, 클립아트 느낌 일러스트, 순수 회색(#888 등 무채색 회색 대신 보라끼 도는 회색 사용: `#8A80A5`).

## 3. 화면 방향 & 좌표계

- **가로 16:9 고정.** 논리 해상도 **1920×1080** 기준으로 모든 좌표를 정의하고, 실제 화면에는 비율 유지 스케일(letterbox)로 맞춘다.
  - `scale = min(innerWidth/1920, innerHeight/1080)`; 게임 루트 요소를 `transform: scale()` 하거나 캔버스 좌표 변환으로 처리. 레터박스 여백은 `--night` 색.
  - 캔버스는 devicePixelRatio 반영 (상한 2).
- **세로로 들면**: 전체 화면을 덮는 오버레이 표시 — "가로로 돌려서 플레이해 주세요 📱↻" + 게임 일시정지(플레이 중이었다면 자동 일시정지, §6.9).
- 모바일에서 첫 터치 시 `requestFullscreen()` 시도(실패해도 무시), `screen.orientation.lock('landscape')` 시도(실패해도 무시).

## 4. 화면 플로우 & 각 화면 스펙

### 4.0 전체 플로우

```
[타이틀] → [홈: 쿠라게 창고] ─┬→ [라이브 준비] → [플레이] → [결과] → (준비 or 홈)
                             ├→ [스카우트]        (Phase 2)
                             ├→ [멤버] → [멤버 상세] (Phase 2)
                             └→ [앨범]            (Phase 2)
```
- **Phase 1에서는** 홈 화면을 생략하고 `타이틀 → 라이브 준비 → 플레이 → 결과` 루프만 구현한다. 화면 전환 구조는 Phase 2 확장을 고려해 라우터 함수(`showScreen(id)`) 하나로 통일.
- 화면 전환 트랜지션: 0.35s, 나가는 화면 `opacity 0 + translateY(12px)`, 들어오는 화면 반대. `prefers-reduced-motion`이면 즉시 전환.

### 4.1 타이틀 화면

- 중앙: 로고 타이포 `마요나카☆아쿠아비트` (핑크→퍼플→민트 그라데이션 텍스트, -3° 회전, 흰색 스트로크 느낌의 drop-shadow), 아래 `MIDNIGHT AQUA BEAT` 레터스페이싱 0.5em.
- 배경: `--night` 바탕에 부유하는 빛 해파리 앰비언트(캔버스 or CSS로 8~12개의 부드러운 발광 원이 느리게 상승, 핑크/민트 반반).
- 버튼: `라이브 시작` (Phase 1) / Phase 2에서는 `문을 연다` → 홈으로.
- 첫 터치/클릭에서 AudioContext를 resume (모바일 오디오 언락).
- 우하단 작게: 버전 표기, `설정` 버튼(§4.6).

### 4.2 라이브 준비 화면 (Phase 1 핵심 UI)

가로를 3열로 분할:

| 좌측 (곡) | 중앙 (센터 멤버) | 우측 (배경 포토) |
|---|---|---|
| 곡 카드: 앨범아트(1:1), 곡명, 밴드명, BPM, 난이도(★), 내 최고기록(점수·랭크·FC 뱃지) | 보유 멤버 가로 스크롤 리스트(원형 얼굴 배지). 선택된 멤버는 크게 + 이름/포지션 표시 | 선택된 센터의 **해금된** 컨셉포토 썸네일 N/R/E 세로 3장. 미해금은 자물쇠 + 해금 조건 툴팁 |

- 하단 바: 노트 속도 선택(1~5, §6.3), `LIVE START!` 버튼(핑크 그라데이션, 대형).
- 센터를 바꾸면 우측 포토 리스트와 미리보기 배경이 즉시 갱신된다.
- 선택 상태(마지막 센터, 속도, 배경)는 저장 데이터에 기억(§10).

### 4.3 플레이 화면 → §6에서 상세 정의

### 4.4 결과 화면

- 배경: 플레이에 쓰인 컨셉포토를 블러(8px) + 딤 40%.
- 좌측 55%: 결과 카드 (데코 카드 스타일)
  - 랭크 대형 타이포(S/A/B/C, 핑크→민트 그라데이션, S일 때만 골드 파티클 5초).
  - 점수 카운트업(0→최종, 1.2초, easeOut). `font-variant-numeric: tabular-nums`.
  - PERFECT/GREAT/GOOD/MISS 4칸 그리드 + MAX COMBO.
  - 풀콤보 시 `FULL COMBO!` 골드 뱃지 + 카드 테두리 골드로.
  - 신기록 시 `NEW RECORD!` 민트 뱃지.
  - 획득 보상 줄: 네온코인 / 팬레터 (+프리즘, 해당 시) — §11 공식.
- 우측 45%: 센터 멤버 표정 이미지(랭크별: S=`fire`, A·B=`joy`, C=`comfort`) + 말풍선에 랭크별 결과 대사(§5의 `resultLines`).
- 버튼: `다시하기` `준비 화면으로` (Phase 2: `홈으로`).

### 4.5 (Phase 2) 홈 / 스카우트 / 멤버 / 앨범

- **홈(쿠라게 창고)**: 수조 배경 + 최애(대표 설정) 멤버 전신 입상. 탭하면 랜덤 인사 대사(컷인 대사 재활용). 수조 안 해파리 수 = `floor(앨범 수집률% / 4)`마리 (0~25). 메뉴 버튼 4개(라이브/스카우트/멤버/앨범) + 재화 3종 표시 바.
- **스카우트**: 연출 시퀀스 — ①게시판에 전단지가 붙는다(0.8s) ②터치하면 전단지가 벗겨지며(0.6s) 뒤에 사인 포스터 = 캐릭터 전신 등장 ③이름/밴드/포지션 텍스트 슬라이드인. 전단지 테이프 색으로 사전 암시: 신규=핑크 테이프, 중복=회색 테이프. 단차/10연 버튼, 확률표 모달(§11.4), 30회 천장 카운터 표시.
- **멤버 상세**: 전신 + 프로필(§5의 서사·갭모에 텍스트 그대로 사용) + 레벨/경험치 바 + 각성 버튼(재료 충족 시 활성) + 컷인 대사 리스트(해금분만).
- **앨범**: 코르크보드에 폴라로이드처럼 붙은 컨셉포토 그리드(랜덤 -3~3° 회전). 탭하면 확대 + 캡션. ★3 해금 시 캡션 아래 비하인드 문단 추가. 상단에 수집률 %.

### 4.6 설정 (모달, 어느 화면에서든)

- 노트 속도(1~5), 배경 딤 강도(§6.7의 기본값 배율 80/100/120%), 판정 오프셋(§6.10), 효과음 볼륨/음악 볼륨(0~100), 데이터 초기화(2단 확인).

## 6. 리듬 코어 상세 (game.js)

### 6.1 플레이 화면 레이아웃 (논리 1920×1080)

```
┌────────────────────────────────────────────────────────────┐
│ [배경: 컨셉포토 1920×1080 cover + 딤 오버레이(§6.7)]           │
│                                                            │
│  좌상단: 점수(우측정렬 숫자) + 진행바(240×10px)               │
│  중앙상단 y=200: 콤보 카운터 (숫자 72px + "COMBO" 레이블)      │
│  중앙 y=320: 판정 텍스트 (PERFECT 등, 44px)                  │
│                                                            │
│        x=560 ┌──┬──┬──┬──┐ x=1360                          │
│              │  │  │  │  │  ← 레인 4개, 각 폭 200px          │
│              │  │  │  │  │     반투명 흰 배경 rgba(255,255,255,.10) │
│              │  │  │  │  │     레인 경계선 rgba(255,255,255,.22) 1.5px │
│   y=930 ━━━━━┷━━┷━━┷━━┷━━━━━ 판정선 (그라데이션 바, 높이 6px, glow) │
│              [D][F][J][K]  ← 키 라벨 (y=960~1020, 데스크톱만) │
│  좌하단: 일시정지 버튼(48px)   우하단: 센터 컷인 존(§6.8)      │
└────────────────────────────────────────────────────────────┘
```

- 노트: 캡슐형(둥근 사각) **186×44px**, 레인 중앙 정렬. 색 = 레인별 `[--pink, --mint, --mint, --pink]`. 위→아래 흰색→레인색 그라데이션 + 흰 스트로크 2.5px + 레인색 glow(shadowBlur 8).
- 스폰 y = **-60**, 판정선 y = **930**. 노트의 화면 y는:
  `y = -60 + (930 - (-60)) * (1 - (note.time - now) / travel)`
- 원근/기울기 없음. 완전 수직 낙하.

### 6.2 시간 기준 (필수 규칙)

- 모든 게임 시간은 **`audioCtx.currentTime - songStartTime`** 을 단일 소스로 사용한다. `performance.now()`나 `Date.now()`로 노트 시간을 계산하지 말 것 (오디오와 어긋남).
- 렌더는 `requestAnimationFrame`, 오디오 예약과 미스 판정은 `setInterval(30ms)` 루프에서 처리.

### 6.3 노트 속도 (유저 옵션)

| 속도 | 1 | 2 | 3 (기본) | 4 | 5 |
|---|---|---|---|---|---|
| travel(스폰→판정선 초) | 2.6 | 2.2 | 1.9 | 1.6 | 1.3 |

### 6.4 판정 윈도우

| 판정 | 윈도우 | 점수 계수 | 콤보 |
|---|---|---|---|
| PERFECT | ±60ms | 1.0 | +1 |
| GREAT | ±115ms | 0.7 | +1 |
| GOOD | ±180ms | 0.4 | +1 |
| MISS | +180ms 초과 경과 | 0 | 리셋 |

- 입력 처리: 탭한 레인에서 `|now - note.time| ≤ 180ms`인 미처리 노트 중 **시간차 절대값 최소**인 것 1개를 판정. 없으면 무시(빈 탭 페널티 없음, 레인 플래시만).
- 미스: 루프에서 `now > note.time + 0.18`인 미처리 노트를 MISS 처리.
- 동시치기(같은 시각 2노트)는 허용되는 채보 형식이며, 멀티터치/동시키로 각각 판정된다.

### 6.5 입력

- 키보드: `D F J K` → 레인 0~3. `keydown`(repeat 무시). `ESC` 또는 `P` = 일시정지.
- 터치/마우스: `pointerdown`, 레인 x 범위(560~1360)를 4분할. **레인 밖 터치도 가장 가까운 레인으로 스냅**하되 판정선 아래쪽(y>700) 영역만. 멀티터치 지원 필수.
- 입력 지연 보정: §6.10 오프셋을 `now`에 가산.

### 6.6 점수 / 랭크

- `노트당 기본점 = 1,000,000 / 총노트수`. 최종점수 = Σ(기본점 × 판정계수), 반올림.
- 정확도 = 최종점수 / 1,000,000. 랭크: `S ≥ 95%`, `A ≥ 88%`, `B ≥ 75%`, 그 외 `C`.
- 풀콤보 = MISS 0 (GOOD 이하 포함돼도 인정).

### 6.7 배경 딤 연출 ("도키메키가 무대를 밝힌다")

- 배경 컨셉포토 위 검정 오버레이의 불투명도가 **그 라이브에서 도달한 최대 콤보 단계**에 따라 내려간다 (콤보가 끊겨도 단계는 유지):

| 도달 콤보 | 0 | 25+ | 50+ | 100+ |
|---|---|---|---|---|
| 딤 불투명도 | 0.55 | 0.45 | 0.35 | 0.22 |

- 전환은 0.8s ease. 설정의 "배경 딤 강도"(80/100/120%)를 곱한다.
- 추가 앰비언트: 콤보 50+부터 배경 가장자리에서 빛 해파리 파티클이 초당 0.5개 상승(최대 12개 동시).

### 6.8 컷인 & 이펙트

- **컷인**: 콤보가 정확히 30/60/100에 도달하는 순간, 우하단에서 슬라이드인(0.25s in → 1.1s hold → 0.25s out). 구성 = 센터 얼굴 이미지(`face_joy`, 128px 원형) + 말풍선(해당 마일스톤의 `cutinLines[i]`). 플레이 방해 금지: 레인 영역과 겹치지 않는 우하단 고정.
- **판정 텍스트**: 중앙 y=320에 0.4s 팝 애니메이션(scale 0.6→1.15→1 + 페이드). 색: PERFECT=핑크→민트 그라데이션, GREAT=핑크, GOOD=골드, MISS=회보라 `#8A80A5`.
- **히트 파티클**: 판정선 위 해당 레인 중심에서 원형 버스트. PERFECT=14개(핑크·민트 혼합, 30%는 ♥ 모양), GREAT/GOOD=8개(레인색). 수명 0.5~0.8s, 중력 400px/s².
- **레인 플래시**: 입력 시 해당 레인에 아래→위 그라데이션 플래시 0.12s.
- **풀콤보 연출**: 마지막 노트 처리 후 MISS==0이면, 결과 화면 전에 1.8s — 화면 전체 플래시(흰 0.15s) → `FULL COMBO!!` 골드 타이포 + 센터 `fcLine` 말풍선 + 파티클 폭발.

### 6.9 일시정지

- 일시정지 시: 진행 중인 오디오 예약을 중단할 수 없으므로 **`audioCtx.suspend()`** 사용, 재개는 3초 카운트다운 후 `resume()`. 일시정지 메뉴: 계속 / 다시하기 / 그만두기(준비 화면으로).
- 화면 회전·탭 이탈(`visibilitychange`) 시 자동 일시정지.

### 6.10 판정 오프셋 캘리브레이션 (설정 내)

- 수동 슬라이더: -100ms ~ +100ms (5ms 단위). `hitTime = rawNow + offset`.
- 자동 측정 모드: 메트로놈 사운드+판정선 점멸을 8박 재생, 유저가 박마다 탭 → 중앙값을 오프셋으로 제안.
- 저장 데이터에 유지(§10).

### 6.11 라이브 종료

- 마지막 노트 시각 + 2.5s 경과 시 종료 → (풀콤보 연출) → 0.5s 후 결과 화면.
- 곡 중도 포기 시 보상 없음, 기록 저장 없음.

## 7. 오디오 엔진 (audio.js)

### 7.1 공통 구조

- `AudioContext` 1개. 마스터 체인: `DynamicsCompressor(threshold -14dB) → GainNode(0.85) → destination`.
- 효과음(판정음 등)과 음악 볼륨은 별도 GainNode로 분리(설정 연동).
- **스케줄러**: 30ms 간격 루프에서 `now + 0.12s` 안에 있는 이벤트를 `songStartTime + event.t`로 예약(lookahead 패턴). 예약 포인터 방식(정렬된 배열 + 인덱스).

### 7.2 신스 보이스 (프로토타입 곡용, 고정 스펙)

| 보이스 | 구성 |
|---|---|
| lead | square 2기(디튠 0.5%) + triangle(+1옥타브, 게인 0.5) → lowpass 4200Hz → ADSR(A 0.012 / 피크 0.16 / 서스테인 0.1) + 딜레이 센드 |
| delay | 딜레이타임 = 박자×0.75(점8분), 피드백 0.3, 톤 lowpass 3000Hz, 센드량 0.22 |
| pad | 코드 구성음 각각 sawtooth → lowpass 1100Hz, 게인 0.035, 어택 0.08 |
| bass | triangle + square(게인 0.25) → lowpass 700Hz, 게인 0.22 |
| kick | sine 150→42Hz 지수 하강 0.11s, 게인 0.5 |
| snare | 노이즈버퍼 → bandpass 1900Hz + triangle 210Hz 바디, 게인 0.25 |
| hat | 노이즈버퍼 → highpass 7500Hz, closed 감쇠 0.02 / open 0.07 |

### 7.3 프로토타입 곡 「하트비트 퍼레이드」 사양

- BPM **132**, 4/4. 시작 오프셋 0.15s. 총 28마디(≈51초).
- 구조: 인트로 4마디(햇+패드+희소 멜로디) → A멜로 8마디 → 빌드 4마디(킥 4분+마지막 마디 스네어롤) → 사비 8마디(풀 드럼+오픈햇) → 아웃트로 4마디.
- 코드: A멜로/사비 = `F–G–Em–Am` 반복, 빌드 = `Dm–Em–F–G`, 아웃트로 = `F–G–C–C` (C장조, 멜로디는 C 펜타토닉 중심 + 사비 최고음 C6).
- **멜로디 이벤트가 곧 채보 노트다** (§8.3). 멜로디는 위 구조·코드 안에서 구현자가 작곡하되: 총 노트 120~170개, 8분음표 이하 금지(16분 없음), 사비가 가장 밀도 높게.
- 레인 배정: 음고 기준 `midi ≤ 72 → 0`, `≤ 76 → 1`, `≤ 79 → 2`, `그 외 → 3`.

### 7.4 정식 곡 (mp3) 파이프라인 — Phase 2

- `songs.js`의 `audio.type === "file"`이면 `<audio>` 아닌 **`AudioBuffer`로 디코드 후 `AudioBufferSourceNode` 재생** (시간 정확도 확보). `songStartTime = ctx.currentTime + 0.2`에 `source.start(songStartTime)`.
- **채보 에디터** (별도 페이지 `editor.html`, Phase 2): mp3 로드 → 파형 표시 → 재생하며 D/F/J/K 입력을 타임스탬프 기록 → 스냅(1/2, 1/4박) 보정 → `notes` JSON 내보내기/불러오기 → 즉시 테스트 플레이.

## 8. 곡/채보 데이터 스키마 (songs.js)

```js
const SONGS = [{
  id: "heartbeat-parade",
  title: "하트비트 퍼레이드",
  bandId: "clione",
  bpm: 132,
  offsetSec: 0.15,
  difficulty: 2,            // ★1~5, 준비 화면 표기용
  audio: { type: "synth" }, // 또는 { type: "file", src: "assets/audio/xxx.mp3" }
  // type==="synth"면 notes는 audio.js의 멜로디에서 생성해 주입한다.
  // type==="file"이면 아래에 명시:
  notes: [ { beat: 8.0, lane: 0 }, { beat: 8.5, lane: 1 } /* … */ ]
  // 노트 시각(초) = offsetSec + beat * (60/bpm)
}];
```

- `lane`: 0~3 (왼→오). `beat`: 곡 시작 기준 박자(소수 허용).

## 11. 경제·성장 수치 (전부 고정값)

### 11.1 라이브 보상 (곡 완주 시)

| 항목 | 공식/값 |
|---|---|
| 네온코인 | `floor(점수 / 2000)` (상한 500) |
| 팬레터(센터에게) | S: 10 / A: 7 / B: 5 / C: 3 |
| 센터 경험치 | `floor(점수 / 5000)` (상한 200) |
| 데일리 보너스 | 그날 첫 라이브는 코인·팬레터 2배 |
| 첫 클리어 | 곡당 1회, 프리즘 +30 |
| 첫 풀콤보 | 곡당 1회, 프리즘 +50 |

### 11.2 레벨

- 최대 Lv 40. `Lv n → n+1` 필요 경험치 = `50 + n × 30`.
- 레벨은 각성 조건으로만 쓰인다(전투력 인플레 없음). 준비 화면에 Lv 표기.

### 11.3 각성 (컨셉포토 해금)

| 단계 | 조건 | 해금 |
|---|---|---|
| ★1 (획득) | 스카우트 또는 초기 지급 | NORMAL 포토, 컷인 대사, 센터 선택 가능 |
| ★2 | 팬레터 30 소모 + Lv 20 이상 | RARE 포토, 결과 보상 팬레터 +1 (그 멤버 센터 시) |
| ★3 | 팬레터 80 소모 + Lv 40 + **그 멤버 센터로 풀콤보 1회** | EPIC 포토, 풀콤보 전용 연출 강화, 앨범 비하인드 텍스트 |

### 11.4 스카우트 (Phase 2)

- 비용: 1회 프리즘 100 / 10연 900. 프리즘 수급: 첫클리어·풀콤보·미션(§11.5).
- 풀: 리더 제외 9명 균등 확률(각 1/9). 중복 획득 시 해당 캐릭터 팬레터 +20.
- 천장: 누적 30회 도달 시 "지명 스카우트권" 1장(원하는 멤버 확정). 카운터는 소모 후 리셋.
- 확률표는 스카우트 화면에서 항상 열람 가능하게(모달).

### 11.5 데일리 미션 (Phase 2)

| 미션 | 보상 |
|---|---|
| 라이브 1회 | 팬레터 5 (대표 멤버에게) |
| 콤보 50 달성 | 프리즘 10 |
| PERFECT 30개 | 코인 100 |

## 12. 아트 에셋 명세 (사용자가 생성해 공급)

### 12.1 공통 스타일 앵커 (모든 이미지 생성 프롬프트 앞에 붙일 것)

```
anime style illustration, cel shading, pastel pink and mint color palette,
kawaii deco sticker aesthetic, high detail, clean lineart
```

캐릭터별 외형 프롬프트는 §5의 `EN 프롬프트` 사용. **같은 캐릭터의 3장은 같은 시드/레퍼런스로 인물 동일성 유지가 최우선.**

### 12.2 파일 규격 (파일명은 §1.2와 정확히 일치해야 함)

| 에셋 | 파일 | 규격 |
|---|---|---|
| 컨셉포토 | `assets/photos/{id}_{n\|r\|e}.webp` | 2560×1440 (16:9), 500KB 이하 권장 |
| 전신 입상 | `assets/chars/{id}_full.png` | 투명 PNG, 세로 2000px 이상 |
| 표정 | `assets/chars/{id}_face_{joy\|fire\|comfort}.png` | 투명 PNG, 800×800 (버스트업) |
| 앨범아트 | `assets/covers/{songId}.webp` | 1024×1024 |
| 밴드 로고 | `assets/logos/{bandId}.png` | 투명 PNG, 가로 1200px |

- 표정 3종 의미: `joy`=기쁨(A/B랭크·컷인), `fire`=열정/환호(S랭크·풀콤보), `comfort`=다정한 위로(C랭크).
- 컨셉포토의 장면 내용은 §5 각 photos의 scene 지시를 따른다.

## 13. Phase별 범위와 완료 기준 (Acceptance Criteria)

### Phase 1 — 수직 슬라이스
구현 범위: 타이틀 / 라이브 준비 / 플레이 / 결과. 클리오네 4인 데이터. 신스 곡 1곡. 저장(기록·설정·선택 기억).
- [ ] 가로 1920×1080 논리 좌표 + letterbox 스케일이 데스크톱/모바일에서 동작
- [ ] 세로로 들면 회전 안내 + 자동 일시정지
- [ ] 신스 곡이 재생되고 노트가 멜로디와 어긋나지 않음 (판정선 도달 = 소리 발음 시점)
- [ ] 판정 4종 + 점수 공식 + 랭크가 §6.4/§6.6 수치와 일치
- [ ] 센터 선택 → 배경 포토(폴백 포함)/컷인 대사/결과 대사가 그 캐릭터로 나옴
- [ ] 콤보 25/50/100에서 배경 딤이 §6.7 표대로 단계 하강
- [ ] 컷인 30/60/100, 풀콤보 연출, 일시정지/재개(3초 카운트다운) 동작
- [ ] 판정 오프셋 설정이 실제 판정에 반영
- [ ] 기록(최고점수·FC)이 localStorage에 저장되고 준비 화면에 표시
- [ ] 이미지 에셋이 하나도 없어도 전 화면이 "의도된 디자인"으로 보임

### Phase 2 — 수집 루프
홈(쿠라게 창고)·스카우트·멤버·앨범 화면, 재화 3종, 각성, 데일리 미션, mp3 파이프라인+채보 에디터, 네온테트라 4인+1곡.

### Phase 3 — 세계의 완성
노크틸루카 4인+1곡, 곡 추가(밴드당 2~3곡), 스토리 모드(에테르노/세라프 메인 플롯), PWA(manifest+SW), 합동 라이브 이벤트.
